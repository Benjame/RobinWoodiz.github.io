介绍一个C++测试框架—Catch

  • Cpp
  • 测试

What is the Catch?

什么是Catch？

Catch stands for C++ Automated Test Cases in Headers and is a multi-paradigm automated test framework for C++ and Objective-C (and, maybe, C). It is implemented entirely in a set of header files, but is packaged up as a single header for extra convenience.

Catch 代表 C++ Automated Test Case in Headers，一种针对C++、Objective-C（并且或许C）多范例自动测试框架。它完全由一系列头文件实现，但为了额外的方便包装成一个单独的头文件。

Key Features

  • Really easy to get started. Just download catch.hpp, #include it and you’re away.

容易学习，仅需下载catch.hpp头文件，把它#inlcude进来即可。

  • No external dependencies. As long as you can compile C++98 and have a C++ standard library available.

没有外部依赖，只要你能编译C++ 98代码并且带有C++标准库即可。

  • Write test cases as, self-registering, functions or methods.

写测试用例，自动注册，函数或方法。

  • Divide test cases into sections, each of which is run in isolation (eliminates the need for fixtures!)

以section的形式分离测试用例，每个分别隔离运行。（排除混合的需要！）

  • Use BDD-style Given-When-Then sections as well as traditional unit test cases.

使用BDD-style Given-When-Then section段 同时也使用传统的单元测试用例。

  • Only one core assertion macro for comparisons. Standard C/C++ operators are used for the comparison - yet the full expression is decomposed and lhs and rhs values are logged.

只有一个核心assertion断言宏用来比较。标准C/C++操作符被用来比较，目前完整的表达式被分解，lhs和rhs值被记入日志。
